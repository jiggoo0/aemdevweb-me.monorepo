# Server-Side Tracking: กุญแจสำคัญสู่ ROAS ที่สูงขึ้น

ในยุคที่ความต้องการความเป็นส่วนตัว (Privacy) สูงขึ้น เบราว์เซอร์อย่าง Safari และ Chrome เริ่มบล็อก Third-party Cookies อย่างเด็ดขาด ส่งผลให้การฝัง Pixel แบบเดิมเสียข้อมูลไปกว่า 30-50%

นี่คือเหตุผลที่ **AEMDEVWEB** บังคับใช้มาตรฐาน **Server-Side Tracking** ในทุกโปรเจกต์ Sale Page และ E-commerce

## 1. CAPI คืออะไร? ทำงานอย่างไร?

**Conversion API (CAPI)** คือการส่งเหตุการณ์ (Events) เช่น การกดซื้อ หรือการกรอกฟอร์ม จาก "เซิร์ฟเวอร์ของเว็บไซต์" ไปยัง "เซิร์ฟเวอร์ของแพลตฟอร์มโฆษณา" (เช่น Facebook, TikTok) โดยตรง

- **ความแตกต่าง:** Pixel ปกติทำงานที่ Browser ของลูกค้า (ซึ่งมักถูกบล็อก) แต่ CAPI ทำงานที่หลังบ้านของเรา ข้อมูลจึงไม่ถูกบล็อกและแม่นยำ 100%

## 2. ประโยชน์ที่ธุรกิจจะได้รับ

1. **แอดแม่นยำขึ้น:** AI ของ Facebook/TikTok จะได้รับข้อมูลที่ครบถ้วน ทำให้หาลูกค้าตัวจริงได้คมขึ้น
2. **ลดค่าแอด (CPA):** เมื่อ Data แม่นยำ AI จะไม่เสียเงินไปกับกลุ่มเป้าหมายที่ไม่ใช่ ลดต้นทุนต่อการทัก (Cost per Message) ได้อย่างเห็นผล
3. **รองรับกฎหมาย PDPA:** การจัดการข้อมูลที่เซิร์ฟเวอร์ช่วยให้เราคัดกรองข้อมูลส่วนตัวก่อนส่งได้อย่างปลอดภัยตามมาตรฐานกฎหมาย

## 3. การปรับแต่งระดับ Specialist

เราไม่ได้แค่ติดตั้ง CAPI แต่เราทำ **Advanced Matching** โดยการส่งข้อมูล Hash ของลูกค้าที่ปลอดภัยไปช่วย AI จับคู่บัญชีผู้ใช้ ทำให้ระบบจำลูกค้าได้แม่นยำแม้จะเปลี่ยนอุปกรณ์เข้าใช้งาน
